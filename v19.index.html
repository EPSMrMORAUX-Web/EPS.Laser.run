🔒 Règles Firebase (CRITIQUE)

Pour protéger les données, copiez ces règles dans la console Firebase → Realtime Database → Règles. Ces règles rendent les passages non modifiables après soumission.

{ "rules": { "sessions": { "$session_id": { "passages": { "$passage_id": { ".read": false, ".write": "!data.exists()" } }, "live": {".read": true, ".write": true}, "presence": {".read": true, ".write": true}, "penalties": {".read": true, ".write": true} } } } }
✅ !data.exists() = écriture une seule fois, modification impossible.
✅ Les passages sont chiffrés AES-256 — illisibles sans le mot de passe enseignant.
✅ Les noms dans Firebase sont anonymisés (initiales seulement).
